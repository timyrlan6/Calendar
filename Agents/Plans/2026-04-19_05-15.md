## Модель данных

### ApplicationUser (расширение)
- `BookingLinkId` — Guid, уникальный, генерируется при регистрации
- `TimeZoneId` — IANA timezone (string)
- 14 полей для рабочих часов (Mon-Sun Start/End, где null = не работает)

### Booking
- `OwnerId` — string (FK на ApplicationUser)
- `GuestName` — string
- `GuestEmail` — string

- `GuestTimeZoneId` — string
- `CreatedAtUtc` — DateTime

Не нужен — всегда 30 минут.

## Принятые решения

### 1. Concurrency (EXCLUDE constraint)
Для гарантии отсутствия пересечений бронирований:
```sql

    "OwnerId" WITH =,
    tstzrange("StartTimeUtc", "EndTimeUtc") WITH &&

```
