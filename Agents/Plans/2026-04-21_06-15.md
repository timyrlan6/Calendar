## Принятые решения

### 1. GUID для публичной ссылки
- **Требование:** GUID генерируется клиентскими средствами C# через `Guid.CreateVersion7()`
- **Конфигурация EF:** `.ValueGeneratedNever()` — чтобы EF не перезаписывал значение из C#
- **Почему Version 7:** sequential GUID, оптимизирован для PostgreSQL (не фрагментирует индекс)

### 2. PostgreSQL 18 и UUID
- PostgreSQL 18.3 поддерживает встроенную функцию `uuidv7()`, но используем клиентскую генерацию через C#

### 3. Timezone
- Брать из браузера пользователя: `Intl.DateTimeFormat().resolvedOptions().timeZone`
- Хранить как IANA timezone string в ApplicationUser.TimeZoneId

### 4. BookingLinkId
- Генерируется при регистрации пользователя (Register.razor)
- URL: `/booking/{guid}`

### 5. TimeOnly поля
- NOT NULLABLE — заполняются при создании пользователя
- Если оба = `00:00` — пользователь не работает в этот день

### 6. Запрос слотов
- LINQ по каждому дню отдельно (SELECT из БД)
- Расчёт в памяти (генерация 30-мин слотов)

## Алгоритм расчёта слотов

```csharp
var (start, end) = date.DayOfWeek switch
{
    DayOfWeek.Monday => (user.MonStart, user.MonEnd),
    DayOfWeek.Tuesday => (user.TueStart, user.TueEnd),
    DayOfWeek.Wednesday => (user.WedStart, user.WedEnd),
    DayOfWeek.Thursday => (user.ThuStart, user.ThuEnd),
    DayOfWeek.Friday => (user.FriStart, user.FriEnd),
    DayOfWeek.Saturday => (user.SatStart, user.SatEnd),
    DayOfWeek.Sunday => (user.SunStart, user.SunEnd),
    _ => (TimeOnly.MinValue, TimeOnly.MinValue)
};

if (start == TimeOnly.MinValue || end == TimeOnly.MinValue)
    return []; // не работает

var tz = TimeZoneInfo.FindSystemTimeZoneById(user.TimeZoneId);
var localStart = date.Date + start.ToTimeSpan();
var localEnd = date.Date + end.ToTimeSpan();
var utcStart = TimeZoneInfo.ConvertTimeToUtc(localStart, tz);
var utcEnd = TimeZoneInfo.ConvertTimeToUtc(localEnd, tz);

var slots = new List<DateTime>();
for (var t = utcStart; t < utcEnd; t = t.AddMinutes(30))
{
    slots.Add(t);
}

return slots;
```

## Concurrency (EXCLUDE constraint)

Будет добавлено в миграции через `migrationBuilder.Sql()`:

```sql
ALTER TABLE "Bookings" ADD CONSTRAINT "no_overlap" EXCLUDE USING gist (
    "BookingLinkId" WITH =,
    tstzrange("StartTimeUtc", "EndTimeUtc") WITH &&
);
```
